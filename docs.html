Importing

Importing frameworks are more efficient as Neuron only converts what it needs and leaves the bloat behind from your CSS and JS frameworks

Media-based programming

When writing your code, you can target specific devices using our mind-blowingly simple media queries and serve to them the HTML, CSS and JS that they need.

To use these media queries, you use:

IF(is{DeviceName})

{do this code}

ENDIF(is{DeviceName})

However, make sure that you replace {DeviceName} with the type of device you’re targeting (we support Mobile, PC, Console, Mac, Windows) and {code} with whatever code you need specifically for that device.

The browser will only be served up content for its device meaning that your pages load a lot faster.

Pro Tip

Build your sites Mobile First and Additively. This means that you build what the smallest device and add in the components that bigger devices will use so that you avoid having unnecessary code.

The Syntax

Neuron has two types of syntax. The first is the block syntax that is to be used for basic page layout. The second is the modifier syntax which is used for bonding, italicising, striking through and hyperlinking content on your page.

Block Syntax

Neuron’s block layout is as follows:

{ELEMENT}
    {content}
/{ELEMENT}

Replace {ELEMENT} with the HTML element that you desire such as SECTION or P or IMG(src: "path/to/pic.png", alt: "alternative text") (more on that last one later).

PSA: Neuron only recognises CAPITAL LETTERS for it’s syntaxes - remember to use them.

Replace {content} with the rest of the content on your page. This can be another block or it can be a modifier.

Modifiers’ Syntax

Modifiers are objects in your page that are specific and small. This is things like bold, italic, strikethrough or an anchor.

Modifiers are written as follows:

{Modifier}("{content}"){Modifier}

where {Modifier} is replaced with a modifier element such as B (note the CAPS) and {content} is text or other content like “This is bolded text”.
